# 实时识别状态与错误提示优化

本 PR 优化 StepFun 实时语音识别过程中的前端状态展示和后端排障日志。

## 前端状态

实时识别过程中会展示更明确的状态：

- 连接实时识别
- 实时通道已连接
- 等待 StepFun 配置
- 正在聆听
- 正在转写
- 正在整理句子
- 实时识别异常
- 实时识别已关闭

## 后端日志

后端补充关键日志：

- 浏览器实时 ASR 连接建立
- StepFun WebSocket 打开
- 发送 `session.update`
- StepFun 会话配置完成
- StepFun 返回错误
- StepFun 关闭状态和原因

## 解决的问题

- 避免实时识别失败时页面静默无反馈。
- 方便通过日志判断问题发生在浏览器连接、StepFun 连接、会话配置还是音频转发阶段。
- 加强对 StepFun 错误结构的解析，优先展示可读的错误消息。
